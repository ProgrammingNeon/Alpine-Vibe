# 🏔️ Alpine Vibe – Landing Page & Lead Management System

Alpine Vibe — це сучасний преміальний лендинг для організації приватних та активних турів Альпами. Проєкт поєднує в собі витончений візуальний стиль **Glassmorphism**, інтерактивні карти та повну автоматизацію збору заявок: дані з форми миттєво зберігаються в базу даних (SQLAlchemy) та надсилаються адміністратору в Telegram-бот (Aiogram 3).

---

## 🌟 Особливості проєкту

* **Естетичний UI/UX:** Дизайн у стилі *Luxury Glassmorphism* з плавними мікроанімаціями, блюр-ефектами (`backdrop-filter`) та адаптивною сіткою.
* **Асинхронний бекенд:** Реалізовано на **FastAPI**, що забезпечує блискавичну швидкість обробки запитів.
* **Паралельна архітектура:** FastAPI сервер та Telegram-бот працюють одночасно в одному асинхронному циклі подій (`asyncio.gather`).
* **Сувора валідація даних:** Захист від спаму та порожніх пробілів на рівні фронтенду (HTML5 regex) та бекенду (**Pydantic v2**).
* **Миттєві TG-сповіщення:** Адміністратор отримує картку ліда в Telegram з inline-кнопкою для швидкого переходу в чат до клієнта (`https://t.me/+phone`).

---

## 🛠️ Технічний стек

* **Frontend:** HTML5, CSS3 (модульна структура стилів), Pure JavaScript (Fetch API).
* **Backend:** Python 3.10+, FastAPI, Uvicorn, Pydantic.
* **Database:** SQLAlchemy (SQLite для розробки, легко мігрує на PostgreSQL).
* **Telegram Bot:** Aiogram 3.x (Асинхронний фреймворк).

# 🚀 Швидкий запуск (Local Deployment)

### 1. Клонування проєкту та підготовка середовища
```bash
git clone https://github.com/ProgrammingNeon/Alpine-Vibe
cd Alpine-Vibe
```

Створіть та активуйте віртуальне середовище Python:
```bash
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для Mac/Linux:
source venv/bin/activate
```

### 2. Встановлення залежностей
```bash
pip install -r requirements.txt
```

### 3. Налаштування оточення (.env)
Створіть файл `.env` у кореневій папці бекенду та додайте дані як в .env-example:
```env
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASS=postgres
DB_NAME=FORMS_BACKEND
API_TOKEN='312312313123123123fsfefefef'
ADMIN_ID = 0000000001
```
⚠️ **Важливо:** Ніколи не завантажуйте файл `.env` на GitHub!


### 4. Запуск серверної частини
Запустіть єдиний скрипт, який підніме веб-сервер та запустить бота:
```bash
python backend/main.py
```

Після цього:
* **FastAPI документація (Swagger UI):** доступна за адресою http://127.0.0
* **Фронтенд:** Відкрийте `index.html` за допомогою Live Server (або будь-якого локального веб-сервера) на порті 5500.

# 🔒 Валідація та Безпека даних
Проєкт захищений від "брудних" даних на двох рубежах:

* **Frontend (JS + HTML5):** Поля телефону приймають виключно цифри через регулярні вирази `oninput`. Кнопка відправки блокується, якщо обов'язкові поля порожні.
* **Backend (Pydantic):** Параметр `str_strip_whitespace=True` автоматично відсікає пробіли. Якщо користувач надішле рядок, який складається лише з пробілів, сервер поверне помилку `422 Unprocessable Content`, захищаючи базу даних від сміття.
